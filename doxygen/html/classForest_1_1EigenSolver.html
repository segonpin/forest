template<int dim>
class Forest::EigenSolver< dim >

Eigenvalue problem solver.

An object eigen_prob of class EigenProb is provided that defines a standard eigenvalue problem. This eigenvalue problem is then solved with any of the methods in this class.

A detailed description can be found in "ref" lattice_sec

template<int dim>
void Forest::EigenSolver< dim >::power_iteration ( SnVector &  phi,
double &  keff 
)

Power iteration method for the eigenvalue problem.

Given the problem

\begin{align*} A x = \lambda x \end{align*}

we use the power iteration to approximate the pair \( (\lambda, x) \).

  • We start with an initial guess \( (\lambda_0, x_0) \), and the first step is to calculate the matrix vector product

    \begin{align*} & y = A x_i \end{align*}

    and update the eigen-pair

    \begin{align*} & \lambda_{i+1} = y^{t} \cdot x_i \ , \quad x_{i+1} = \frac{y}{\lambda_{i+1}} \end{align*}

  • Now we estimate the different errors

    \begin{align*} & e_{\lambda} = \frac{\| \lambda_{i+1} - \lambda_{i} \| }{\lambda_{i}} \ , \quad e_{x} = \| x_{i+1} - x_{i} \| \end{align*}

    and decide the stopping criteria. If this criteria is not fulfilled, we increase the iteration index and start again.

Solve the eigenvalue problem with SLEPc, by default using the Krylov-Schur method.

Call SLEPc eigenvalue problem solve to solve.

SLEPc need its own PETSc vector so a full vector copy must be performed. This copy was tested to cost less than 0.1% of the total computational time.
